# 数字员工 App UI 设计规范与全页面清单

> 文档状态：视觉方向已确认基线
> 版本：v0.2
> 更新日期：2026-07-21
> 需求基线：《需求设计.md》v0.9
> 技术基线：《技术设计.md》v0.8
> 设计目标：以 Apple 官方 Human Interface Guidelines 为基础，建立“原生生产力界面 + 分层 2D 互动办公室叙事”的统一体验。

---

## 1. 设计目标与成功标准

本产品不是把聊天窗口换成员工头像，也不是把公司模拟游戏套在 AI 外面。界面必须同时完成三件事：

1. 用户能像使用成熟 iPhone 生产力工具一样，快速提交、查看、复制、分享和修订真实成果；
2. 用户能在固定高位、娃娃屋式构图的互动办公室里，看见员工的工作与生活状态，并理解“我的问题就是公司的业务”；
3. 所有完成、进度、收入、支出和人事变化都对应服务端真实状态，不用场景演出伪造 AI 能力。

视觉验收成功标准：

- 第一次使用者在 30 秒内能找到“新建任务”；
- 登录后默认进入办公室，顶级导航固定为办公室、任务、公司、汇报、我的；
- 用户不看说明也能区分“员工正在工作”“需要我补充”“结果可查看”和“员工正在休息”；
- 正式结果页的阅读、选择、复制、分享和 Dynamic Type 体验不弱于常规 iOS 生产力 App；
- 场景资源失败、减少动态效果、VoiceOver、弱网和供应商故障时，核心任务仍可完成；
- 所有视觉稿使用同一套导航、字号、间距、语义色、状态词和组件规则。

## 2. Apple 设计基线

设计与实现以当前 Apple HIG 和 iOS 官方 UI Kit 为事实基线，不逐像素模仿某个 Apple App，也不使用未经验证的“苹果风”装饰。

### 2.1 信息层级与导航

- React Native 业务页使用系统导航栏、列表、分组表单、Sheet、Menu、Toolbar 和 Tab bar 的熟悉行为；
- 登录后默认一级首页为“办公室”；顶级 Tab 固定为“办公室、任务、公司、汇报、我的”，位置和名称不随状态改变；
- “新建任务”是办公室控制栏和任务页工具栏中的明确操作，不得伪装成 Tab；
- 办公室与 Tab bar 同属一级结构，不进入隐藏导航体系；场景内控制栏保留新建任务、任务调度、会议、日报 / 秘书摘要、场景等价列表和减少动态效果；
- 场景采用“全景 → 房间聚焦 → 对象详情”三级交互；对象详情、任务创建、筛选和员工快捷信息优先使用 Sheet，长文、复杂表单和正式结果使用独立原生页面；
- 破坏性、付费、解雇、数据删除和无法撤销操作使用清晰确认，不依赖手势作为唯一入口；
- P0 不提供自由旋转、摇杆、自由漫游或任意家具摆放，拖拽不得作为任何核心操作的唯一入口。

### 2.2 字体、图标与触控

- 使用 iOS 系统字体与 Dynamic Type 样式，不把视觉稿中的固定字号直接当作实现值；
- 正文字号以系统 Body 为基准；图片中的最小辅助文字仍需在正常尺寸清晰可读；
- 图标优先使用 SF Symbols，与文字标签共同表达关键状态，不使用 Emoji 代替产品图标；
- 常用控件以 44 × 44 pt 为默认可点击区域；较小可见图形必须用透明内边距扩展点击区；
- 颜色不是唯一状态信号，必须同时提供图标、文字或形状；
- 支持文字放大至少 200% 的布局重排，避免固定高度截断答案和人事信息。

### 2.3 色彩、材质与品牌

- 业务 UI 使用系统语义背景、标签、分隔线、填充和状态色，自动适配深浅色及提高对比度；
- 品牌主色用于主操作、选中态和少量关键强调，不把每张卡片都染成品牌色；
- 品牌气质为可信、温暖、聪明、有一点幽默，但不幼稚；
- 互动办公室采用非像素、明亮温暖卡通风格：清晰轮廓、柔和木色、米白、雾蓝和植物绿；卡通员工与原生覆盖层保持清晰层级；
- 固定高位、娃娃屋式构图通过前景、中景、背景、遮挡和比例建立空间感，不使用技术类型标签定义视觉；
- 员工关键姿态应面向或以 3/4 角度面向用户，避免长期背对；
- Liquid Glass / 系统材质只用于导航、工具和临时浮层等层级表达，不在长列表与正文背景上堆叠自制模糊玻璃；
- 正式稿使用 Apple 当前官方 UI Kit 校准安全区、系统控件、导航与符号尺寸。

### 2.4 动效、进度与反馈

- 动效说明层级与因果，不用持续漂浮、弹跳和粒子制造“高级感”；
- 已知长度的下载、解析、导出可显示确定进度；AI 生成等无法量化的过程使用不确定进度和真实阶段描述，不显示虚构百分比；
- 加载时尽快显示可用页面骨架、已有任务和场景等价列表，让用户继续浏览其他内容；
- 关键任务可取消时提供取消；不可安全取消时解释正在保存或结算；
- 开启“减少动态效果”后，用淡入、状态替换和静态聚焦取代长路径移动、视差、缩放与持续演出；
- “暂停新任务调度”只停止未执行任务的自动分配，不暗示已发出的模型请求被暂停；“减少 / 暂停办公室演出”只影响客户端表现；不得出现模糊的“全部暂停”；
- 触觉反馈只用于明确提交、成功、警告和选择，不为普通滚动或持续进度重复触发。

### 2.5 无障碍与场景等价入口

- VoiceOver 阅读顺序先任务 / 员工名称，再状态、关键信息和操作；
- 场景中的每名员工、房间、任务标签、任务道具和协作气泡，在场景等价列表中都有可聚焦、可操作的等价行；
- 关键音效同时提供视觉提示；关键状态同时使用文字以及图标、形状或纹理之一，不只靠颜色；
- 不要求用户精确点击小人物：可见对象使用扩展点击区，并提供员工 / 房间 / 任务列表；
- 标签优先级固定为：等待老板 / 阻塞与失败 > 汇报就绪 > 进行中主要任务 > 人事与关系事件 > 装饰状态；超出场景容量的标签进入聚合摘要；
- 场景资源失败、过热降档或资源未下载时，自动切换到场景等价列表，禁止用不可操作截图冒充场景。

## 3. 设备与版式基线

- 主视觉稿画布：iPhone 竖屏 390 × 844 pt；同时验证 375 × 667 pt 和 430 × 932 pt；
- 页面内容尊重顶部与底部安全区；键盘弹出时主操作不得被遮挡；
- 业务页面采用 4 pt 基础网格，常用间距为 8 / 12 / 16 / 20 / 24 / 32 pt；
- 列表和表单水平安全边距默认 16 pt；正文阅读区可按内容类型使用更舒适的 20 pt；
- 办公区采用固定前方高位、娃娃屋式构图并始终位于视觉中心；P0 不自由旋转；
- 全景突出 4—6 人，当前楼层最多 8 人；同时移动最多 3 人，完整任务标签最多 2 个，对话或协作气泡最多 1 个；
- 其他员工按房间、楼层或秘书摘要聚合；场景控制栏不得遮挡角色头部、标签和主要通道；
- 员工同一时间只显示一个权威位置、一个主要动作和一个 `primary_visual_task`；
- 横屏不是首版核心体验，不得以横屏才能看清办公室或完成操作。

## 4. 信息架构

```text
启动 / 登录 / 公司初始化
└── 顶级导航（登录后默认：办公室）
    ├── 办公室
    │   ├── 全景 / 房间聚焦 / 对象详情
    │   ├── 新建任务 / 任务调度 / 会议 / 日报
    │   └── 场景等价列表 / 减少动态效果
    ├── 任务
    │   ├── 新建 / 补充 / 进度 / 正式结果 / 修订
    │   └── 项目与版本
    ├── 公司
    │   ├── 员工与完整 HR
    │   ├── 财务与工资
    │   └── 资产、设备与办公室扩建
    ├── 汇报
    │   ├── 秘书摘要 / 完成收件箱
    │   ├── 审批 / 异常 / 日报
    │   └── 通知
    └── 我的
        ├── 订阅与算力额度
        ├── AI 数据同意与隐私
        └── 账号、通知、无障碍与数据管理
```

“新建任务”是办公室和任务页中的明确操作，不属于顶级导航。复杂详情、编辑表单、审批和正式结果进入原生页面 / Sheet。

## 6. 互动办公室员工状态视觉字典

| 业务事实 | 允许的角色表现 | 场景 UI | 禁止表现 |
|---|---|---|---|
| `available` | 阅读、整理桌面、短聊天、茶水、伸展 | “可接任务”或不打扰的状态点 | 持续敲键盘冒充工作 |
| `assigned` | 接文件、看任务卡、走向工位 | 员工名 + “刚收到任务” | 已完成标记 |
| `queued` | 阅读需求、等待设备 / 会议室 | “正在排队 / 准备资料” | 虚构百分比 |
| `executing` | 坐下工作、查资料、使用任务对应道具 | 真实阶段短句、不确定进度 | 用动作速度暗示剩余时间 |
| `tool_running` | 走向资料室、数据屏、会议桌 | 显示真实工具名称或“正在解析文件” | 没有工具事件却演同类动作 |
| `waiting_user` | 停止工作、举手 / 走向秘书 | “需要你补充 1 项信息” | 表现为拒绝、怠工或失败 |
| `retrying` | 整理资料、短暂等待、重新坐下 | “连接不稳定，正在重试” | 红色恐慌、责怪员工 |
| `output_ready` | 整理文件、起身、走到汇报点 | “汇报已准备好”主操作 | 结果未持久化就汇报 |
| `completed_seen` | 回到工位、短暂庆祝或正常活动 | 结算卡在结果页展示 | 无限庆祝阻塞操作 |
| `break` | 茶水、聊天、短娱乐、休息 | “休息中”，有任务仍可派发 | 误标为离线或不可用 |
| `off_duty` | 离开、空工位、关闭主灯 | 加班 / 电话召回选择 | 让 AI 因下班停止执行 |
| `overtime` | 夜间局部照明、专注工作 | 明确加班原因与当前任务 | 用疲劳惩罚降低答案质量 |
| `phone_recall` | 接听、通勤占位、进入办公室 | 已联系 / 赶来 / 已到岗 | 未到岗就显示现场工作 |
| `leave` | 空工位、日历标记 | 请假时间、代理人、交接 | 员工仍随机出现在办公室 |

场景对象规则：

- 员工同一时间只有一个权威位置和一个主要动作；多任务只选择一个 `primary_visual_task` 驱动标签、姿态和道具；
- 员工、任务标签、任务道具、房间和协作气泡分别映射到唯一对象 ID；点击后按 B04—B08 打开原生快捷卡或 Sheet；
- 语义槽位定义工位、门口、会议位、汇报位、茶水位等合法位置；员工只沿预设路径切换槽位；
- 对话、会议、交接与汇报必须由真实事件触发，事件结束或状态失效时立即撤下；
- 标签遵循第 2.5 节优先级和数量上限，不用颜色作为唯一状态信号；
- 关键姿态面向或以 3/4 角度面向用户，背面移动结束后应回到可读姿态。

## 7. 核心交互原型必须覆盖

完整 UI 图之后，交互原型至少串联以下路径：

1. 首次进入 → 创建公司 → 选择员工 → 第三方 AI 同意 → 登录后默认进入办公室；
2. 办公室全景 → 房间聚焦 → 点击员工 / 任务标签 / 任务道具 / 房间 / 协作气泡 → 对象详情；
3. 办公室控制栏 → 新建任务 → 员工接单 / 工作 → 需要补充 → 继续执行 → 汇报就绪；
4. 任务调度 → 调整负责人 / 优先级 → 暂停新任务调度；验证已执行模型请求未被伪装暂停；
5. 真实事件触发会议 → 会议编排 / 复核 → 打开任务；无事件时不出现会议或协作气泡；
6. 日报 / 秘书摘要 → 多任务筛选 → 打开结果 → 处理审批；
7. 查看汇报 → 阅读正式结果 → 查看来源 → 退回修改 → 新版本 → 虚拟收入结算；
8. 招聘 → 入职 → 考勤 → 发薪 → 绩效 → 晋升 / 调岗 → 离职 / 解雇；
9. 任务收入 → 公司余额 → 购买设备 → 选择预设槽位 → 办公室扩建；
10. 订阅购买 / 恢复 → 算力用量 → 额度提醒 / 不足；
11. 非工作时间派工 → 加班或电话召回 → 夜间工作 → 汇报；
12. 场景资源失败 / 过热 → 场景等价列表 → 继续任务与打开结果；
13. 减少 / 暂停办公室演出 → 动作静止或简化 → 服务端任务状态继续更新；
14. 弱网、App 后台、进程被杀 → 回到同一权威任务和场景状态。

## 8. 视觉交付批次

为了保证“全部页面”不是一组风格互相冲突的单张图，视觉工作按以下不可跳过的顺序进行：

1. 黄金视觉基线：基于已确认的非像素、明亮温暖卡通、固定高位娃娃屋方向，先完成办公室主场景和关键汇报状态，验证品牌、分层 2D、信息密度与 Apple 原生感；
2. 同方向受控变体：只允许在相同构图、透视、角色比例和美术语言内比较色温、信息密度、标签强度等有限变量；评审后冻结一套黄金视觉，不重新开启三套互不相干的风格探索；
3. 核心闭环稿：A01—A08、B01—B20 / B23—B24、C01—C12、D01—D06；
4. 公司经营稿：E01—E14、F01—F06；
5. 商业与设置稿：G01—G12、H01—H07；
6. 扩展状态稿：B21—B22、C13、E15、F07—F08、G13、H08；
7. 交互原型与视觉 QA：按第 7 章逐条跑通，检查安全区、字号、对比度、点击区、状态真实性和跨页一致性。

每个页面至少交付默认状态；存在加载、空、错误、禁用、选中、完成或破坏性确认时，必须补充对应状态。员工的工作与非工作状态、三级交互、对象点击、控制栏、任务调度、会议、日报和场景等价列表必须在 B 区场景稿中可视化。

## 10. 当前冻结结论

- 视觉遵循 Apple 官方结构、控件习惯、无障碍与反馈原则；办公室采用非像素、明亮温暖卡通、清晰轮廓、柔和木色 / 米白 / 雾蓝 / 植物绿的品牌风格；
- 办公室是登录后默认一级首页，顶级导航固定为办公室、任务、公司、汇报、我的；新建任务是明确操作而非 Tab；
- 办公室在 React Native 内采用固定前方高位、娃娃屋式构图的分层 2D 互动场景，竖屏优先；P0 不自由旋转、不使用摇杆、不允许任意家具摆放；
- 首版办公室固定为临时工作点、小型独立办公室和标准公司楼层三套基础模板；三套共享角色比例、透视、美术语言和交互层级，并完整覆盖扩建、降级和搬迁；
- 交互固定为全景、房间聚焦、对象详情三级；用户以老板视角直接点选对象，不控制老板角色移动；
- 员工、工位、房间、任务、交接、会议与汇报可见可点，使用语义槽位和预设路径；复杂详情和正式结果使用原生页面 / Sheet；
- 全景重点 4—6 人、当前楼层最多 8 人、同时移动最多 3 人、完整任务标签最多 2 个、对话或协作气泡最多 1 个；其他员工聚合；
- 同一员工只有一个权威位置、主要动作和 `primary_visual_task`；会议、对话、交接与汇报必须来自真实事件，会议不默认表示多个独立 Agent；
- 标签有明确优先级，状态不只依靠颜色；场景等价列表、屏幕阅读与减少动态效果覆盖所有关键操作；
- 不存在“全部暂停”；仅提供边界清晰的“暂停新任务调度”和“减少 / 暂停办公室演出”，前者不伪装暂停已执行模型请求；
- 正式结果是权威 `WorkProduct`，员工口吻只是一段 `PersonaReport`；
- 当前按已确认方向完成一套黄金视觉，并只在同一构图、透视、角色比例和美术语言内制作受控色彩 / 密度变体；黄金视觉签字后再生成本文全部页面和状态，禁止重新开启互不相干的风格探索或一页一个风格；
- 所有 UI 图与交互最终仍需对照 Apple 当前官方 UI Kit、真机可读性和实现约束复核。
